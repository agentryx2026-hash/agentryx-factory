# 11 - The Launch Roster: Minimum Viable Factory Floor

## 1. Initial Launch Philosophy
To launch a functional autonomous software factory, attempting to build 15 hyper-specialized agents immediately leads to complex failure chains. We must start with a core "Baseline Team" that mimics a standard agile squad. Once this team's communication protocols are perfected, scaling to specialized departments (e.g., dedicated database agents, graphics agents) becomes trivial.

## 2. The Core 5-Agent Launch Team

### Agent 1: The Product/Project Manager ("Alice")
* **Role:** Intake, Translation, and Strategy.
* **Responsibilities:** Receives the raw webhook from n8n (e.g., a messy user bug report), translates it into a strict JSON development spec, and breaks it down into sequential sub-tasks.
* **Launch Model:** `gemini-1.5-flash` or `gemini-1.5-pro` (Reasoning and parsing text is cheap and fast).

### Agent 2: The Junior Developer / Grunt Work ("Charlie")
* **Role:** High-volume code generation.
* **Responsibilities:** Writing boilerplate, standard React components, basic CSS layout modifications, and updating documentation strings.
* **Launch Model:** `gemini-1.5-pro` (Strong at standard coding logic, mid-tier cost).

### Agent 3: The Senior Architect ("Henry")
* **Role:** High-complexity problem solving.
* **Responsibilities:** ONLY wakes up to debug cryptic errors, architect database migrations, or handle core algorithmic rewrites where "Charlie" failed three times in a row.
* **Launch Model:** `gemini-3.1-pro` (Highest intelligence, expensive, used sparingly).

### Agent 4: The Quality Assurance Reviewer ("Ralph")
* **Role:** Security and Regression Testing.
* **Responsibilities:** Reads the code generated by Charlie and Henry. Looks maliciously for security loopholes, unhandled exceptions, and poor optimization. Sends it back if it fails.
* **Launch Model:** `gemini-1.5-pro` (Great at spotting anomalies via RAG).

### Agent 5: The Site Reliability Operator ("Ops-Bot")
* **Role:** Git and Environment Execution.
* **Responsibilities:** Strictly handles the final stage. Commits code, writes the commit message, and pushes the Pull Request to GitHub. No coding logic required.
* **Launch Model:** N/A or `gemini-1.5-flash` (Can usually be completely deterministic bash scripts with no expensive LLM usage).

## 3. UI Update Implementation
To support this "Launch Team," the dashboard configuration requires an update. Instead of just pasting an API key, the Admin section must now allow the developer to manually assign which specific API model string is given to Alice, Charlie, and Henry.
