# Module — Verify Portal Integration

The Verify portal is a **separate application** (`verify-stg.agentryx.dev`, currently in production for HireStream). The factory **integrates with** it; doesn't subsume it.

## Boundary contract (sketch — finalized in Phase 9)

### Factory → Verify (publish)

When a phase / artifact bundle is ready for human verification, the factory POSTs:

```
POST {VERIFY_URL}/api/projects/{project_id}/builds
{
  "build_id": "...",
  "version": "...",
  "artifacts": {
    "preview_url": "...",      // deployed staging instance
    "test_cases": [...],        // auto-generated by Tuvok
    "screenshots": [...],
    "release_notes": "...",
    "agent_trace_url": "..."    // Langfuse link
  }
}
```

Verify creates a review queue, notifies assigned reviewers (Hermes — Phase 10).

### Verify → Factory (feedback)

When a reviewer marks an item Pass / Partial / Fail, Verify webhooks back:

```
POST {FACTORY_URL}/api/verify/feedback
{
  "build_id": "...",
  "requirement_id": "...",
  "decision": "pass" | "partial" | "fail",
  "comments": "...",
  "screenshots": [...],
  "reviewer": "..."
}
```

Factory routes feedback to the appropriate fix-cycle agent (likely Spock for code fixes, Tuvok for additional test coverage).
